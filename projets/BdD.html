Voici une matrice de covariance, le résultat d'un travail sur des données publiques et qui prouvent un lien de corrélation entre la longueur du nom des établissements et le taux de réussite de leur élèves.

Compétence

  • Gérer des données de l'information
  • Travailler dans une équipe informatique

Description

Ce projet, réalisé durant la première année de BUT Informatique, portait sur la création, le peuplement et l'exploitation d'une base de données, en utilisant des outils mathématiques, et se focalisait sur les établissements scolaires, leur position géographique et le taux de réussite de leurs élèves.

Phase 1 : Création de la base de données

La première étape consistait à créer la base de données en suivant un diagramme UML fourni. Il nous fallait rédiger un script SQL pour la création de la base de données, en respectant les cardinalités indiquées dans le diagramme UML. Ce script devait définir les tables, les clés primaires et étrangères, ainsi que les contraintes nécessaires pour assurer l'intégrité des données.

Phase 2 : Peuplement de la base de données

Pour cette phase, nous avons reçu des données publiques réelles sous forme de fichiers .csv. Notre tâche était de créer un script SQL qui importe ces données dans des tables temporaires à l'aide de la commande Wbimport. Ensuite, nous devions insérer les données dans les tables principales de la base de données. Cette étape nécessitait parfois l'utilisation de jointures, surtout lorsque les données étaient réparties entre plusieurs tables temporaires.

Phase 3 : Exploitation des données

La dernière phase consistait à analyser les données pour identifier des corrélations potentielles. Nous avons sélectionné cinq séries de données et vérifié s'il existait des corrélations entre elles, puis élaboré des hypothèses basées sur les résultats obtenus. Voici les étapes détaillées de cette phase :

  1. Création d'une vue : Nous avons créé une vue dans la base de données contenant les séries de données d'intérêt.
  2. Exportation de la vue : Utilisant la commande Wbexport, nous avons exporté cette vue en fichier .csv.
  3. Analyse en Python : Nous avons importé le fichier .csv en Python pour effectuer une analyse plus approfondie. Cela incluait :
    • La représentation graphique des données pour visualiser les tendances.
    • La création d'une matrice de covariance pour évaluer les relations entre les variables.
    • La réalisation d'une régression linéaire multiple pour comprendre comment plusieurs variables explicatives peuvent influencer une variable endogène.

Résultats et apprentissages

Ce projet m'a permis de comprendre le processus complet de création et d'exploitation de statistiques. J'ai acquis des compétences pratiques en SQL pour la gestion de bases de données, en importation et exportation de données, et en analyse statistique à l'aide de Python.
